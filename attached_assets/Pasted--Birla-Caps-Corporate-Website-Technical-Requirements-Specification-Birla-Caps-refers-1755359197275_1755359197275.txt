### 3.3 Database Schema (Key Tables)
```sql
CREATE TABLE investor_documents (
    id UUID PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    type VARCHAR(50) CHECK (type IN ('ANNUAL_REPORT','QUARTERLY_RESULT','OFFER_DOC','SUSTAINABILITY_REPORT','ESG_PROFILE')),
    fiscal_year INT NOT NULL,
    s3_key VARCHAR(512) NOT NULL,
    downloads INT DEFAULT 0,
    version SMALLINT DEFAULT 1,
    metadata JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ
);

CREATE TABLE grievances (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    subject VARCHAR(100) NOT NULL,
    description TEXT,
    status VARCHAR(20) DEFAULT 'OPEN' CHECK (status IN ('OPEN','IN_PROGRESS','RESOLVED','ESCALATED')),
    assigned_to UUID REFERENCES users(id),
    resolution_deadline TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE policies (
    id UUID PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    effective_date DATE NOT NULL,
    version SMALLINT DEFAULT 1,
    s3_key VARCHAR(512) NOT NULL,
    change_history JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

(Added policies table to support version-controlled policy management.)

## 4. **Security Requirements**
1. **Data Protection**:
   - AES-256 encryption at rest (RDS)
   - TLS 1.3 for data in transit
   - PII masking in logs

2. **Access Control**:
   - Role-based permission matrix
   - IP allowlisting for admin panel
   - Session fingerprinting

3. **Vulnerability Management**:
   - OWASP Top 10 safeguards
   - Quarterly penetration tests

4. **Compliance**:
   - SEBI electronic record guidelines
   - GDPR/CCPA compliance hooks
   - Audit trail retention (7 years)
   - IRDAI/NHB grievance redressal integration
